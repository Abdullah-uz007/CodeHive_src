# CodeHive

CodeHive is an AI-assisted Python project generator. A user request is passed
through planner, research, architecture, code, and fix agents. Each run creates
a separate project folder under `generated_projects/` and also refreshes
`generated_code/` as a copy of the latest output.

## What It Builds

The UI lets you choose a project type before generation:

- Flask API
- Streamlit App
- CLI Tool
- Machine Learning Script
- General Python Project

The selected type is passed into every agent so the generated architecture,
dependencies, and runnable entry file are better matched to the request.

## Setup

1. Create and activate a virtual environment.
2. Install CodeHive dependencies:

   ```powershell
   pip install -r requirements.txt
   ```

3. Copy `.env.example` to `.env` and add your OpenAI API key.
4. Run the Streamlit UI:

   ```powershell
   streamlit run ui.py
   ```

You can also run the CLI:

```powershell
python main.py
```

## Dashboard

The Streamlit UI shows:

- planner output
- research output
- architecture output
- generated files
- dependency install status
- runtime status
- fix attempts
- generated project folder contents

## Dependency Installation

By default, CodeHive does not install dependencies generated by the AI. In the
UI, enable "Install generated dependencies" when you want it to create a `.venv`
inside the generated project folder and install that project's requirements
there.

For CLI usage, enable it with:

```powershell
$env:CODEHIVE_AUTO_INSTALL = "1"
```

This keeps generated dependencies out of your main CodeHive environment.

## Packaging Notes

Do not include `.env`, `.git`, `venv/`, `.venv/`, `__pycache__/`, or generated
project `.venv` folders when sharing the project.
